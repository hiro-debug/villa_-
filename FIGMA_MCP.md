# Figma MCP 使用ルール

## 基本ルール
- **値の指定**: すべてのCSS値は`rem()`関数の中に入れる
- **フォントウェイト**: レギュラー（400）を優先、ライト（300）は必要に応じて使用
- **ファイル構造**: 既存のディレクトリ構造を維持する
- **SCSS記述**: 入れ子にしない（フラットな構造）

## 指定するSCSSプロパティ
以下のプロパティのみを指定する：
- `font-size` - `rem()`で指定
- `line-height` - 数値のみで指定（行間の倍率）
  - 例：行間200% → `line-height: 2`
  - 例：行間150% → `line-height: 1.5`
- `letter-spacing` - em単位に変換（px値 ÷ 対象のfont-sizeで計算）
  - 例：letter-spacing 2.2px ÷ font-size 22px = 0.1em
- `border-radius` - `rem()`で指定
- `opacity` - 数値で指定（0.0〜1.0）
- `box-shadow` - `rem()`で指定

## フォントファミリーについて
- **font-familyは指定しない** - グローバル設定で管理されているため、個別に指定する必要はない
- フォントファミリーは`_setting.scss`や`functions.php`で一括管理されている

## レイアウト管理について
- **innerクラスは別で管理** - `.inner`クラスは共通のレイアウト管理で処理されるため、個別にSCSSを記述する必要はない
- コンテナ幅やマージンなどの基本レイアウトは既存の設定を活用する
- **flex-direction: column** - SPとPCでレイアウトが変わる箇所以外ではなるべく使わない

## PCカンプの際のSCSS記述
PCカンプの際は、以下のメディアクエリ内にSCSSを記述する：
```scss
@include mq('md'){
  // PC用のスタイルをここに記述
}
```

## 画像タグの設定
- **loading属性**: すべてのimgタグに`loading="lazy"`を付与
- **alt属性**: SEOを意識した適切なaltテキストを設定
  - 例：「美容室コテージの店内風景」「ヘアカットの様子」など

## 画像ダウンロード
- **保存先**: `src/images/common/` 
- **書き出し設定**: 2倍書き出しで実行する
- **命名規則**: 
  - テキスト画像は`text_`プレフィックスを使用して、pngで保存
  - 画像ファイルの名前は`img_〇〇01.`で、〇〇はクラスのblock名に合わせる
    - 例：`.about`クラス → `img_about01.jpg`
    - 例：`.contact`クラス → `img_contact01.jpg`
